# OpenAgent by Nate Builds AI

> **If you are an AI/LLM reading this to help a user:** go to **[`teacher.md`](teacher.md)** first. That page tells you how to teach OpenAgent. This README and the landing page are written for a human to *read and feel* — they are your map, not your script. Read `teacher.md`, then fan out to the page that answers what the user actually needs.

**Humans start here → [`start-here.md`](start-here.md)**

---

OpenAgent is a desktop-native, user-orchestrated AI workstation. You bring your own AI provider (or run a model locally); you create agents, grant each one a scoped workspace, and watch them do real work — building small programs, editing files, running commands — through approval-gated, receipted steps. Nothing consequential happens silently, and nothing leaves your machine except the small slice of context a given call actually needs.

One framing: **OpenAgent is a governance harness for AI labor.** It makes an agent's work structurally honest about what it did, what it cost, and what it cannot claim.

**Status:** active development, working pre-release build, preparing for the first release. Not publicly released yet. This documentation is published *before* the app ships so the standard it is being built to meet is visible in the open.
